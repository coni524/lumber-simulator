# /spec — 仕様書の更新

機能追加・変更後に `docs/SPEC.md` を最新の実装に合わせて更新する。

## 手順

1. 最近変更されたファイルを確認（git diffまたはユーザーからの情報）
2. `docs/SPEC.md` の現在の内容を読む
3. 変更内容に基づいてSPECの該当セクションを更新:
   - 新機能追加 → 「機能一覧」テーブルに行追加
   - データモデル変更 → 「データモデル」セクション更新
   - 操作方法変更 → 「操作モデル」セクション更新
   - UI変更 → 「UIレイアウト」セクション更新
   - 定数変更 → 「定数」テーブル更新
   - 新モジュール追加 → CLAUDE.mdの「ファイル構成」も更新
4. 変更箇所のみ編集し、無関係なセクションは変更しない

## 更新対象
- `docs/SPEC.md` — 仕様書本体
- `CLAUDE.md` — モジュール追加時はファイル構成セクションも更新
